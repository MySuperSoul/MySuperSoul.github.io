周赛-246

001-句子中的有效单词数

这道题就是一个写起来比较复杂的模拟题。第一步，就是将输入的句子给拆出来，变成一个个的token，然后对每个token去做判断。在每个token当中进行判断，对于token当中的每个字符 $c$ 有四种情况：

  • c >= ‘0’ && c <= ‘9’, return false

  • c >= ‘a’ && c <= ‘z’, continue

  • c == ‘-‘

    • 首先看之前是不是见过连字符，如果已经见过了，说明后面再出现一定是invalid的。
    • 再判断连字符的位置，如果它是在首尾，一定不满足左右都是小写字母的条件，返回false就好。
    • 然后判断它左右两边是不是小写字母。

  • c == ‘!’ ‘.’ ‘,’

    • 这个只需要判断一下位置就好了。

判断函数搞定之后，就可以得到题目需要的数量了，每个token判断一下就完了。

003-统计最高分的节点数目

这题仔细分析一下就可以得知，当去除掉一个节点的时候，计算得到的得分就是 (左子树节点数 * 右子树节点数 * (总节点数 - 该节点子树节点数))。

所以我们的目标就是给每个节点计算一下它的左右子树的节点数，并且记录下来就好了。然后就是dfs遍历每个节点去计算一下当前的分数，不过需要注意的是，如果上面的计算方式某一项为0的时候，这个时候应该是不把它算进公式里，这里我们也相当于乘1。在算结果的时候可能会爆int，所以我们换成long long就好了。

最后将结果存到一个map当中，这个map的最后一项就是我们要的答案。

004-并行课程 III

这题跟上面这题比较像的思路。首先第一步是找到root节点，root节点是那些出度为0的节点，也就是在relations当中没有在第一项中出现的那些节点。不过这个root节点很有可能不止一个，所以我们都存下来。

对于每个root节点，就只需要往下DFS去计算每个前驱节点完成所需要的最短时间，然后计算节点间的所有最短时间的最大值（因为必须要所以前置节点全部完成才能开始当前节点任务），加上完成当前节点任务的时间，组成当前节点的最短时间。

一直传到root节点，然后在计算所有root节点之间的最大值即可。这里加了一个优化，就是如果一个节点之前已经被计算过了，那么它的结果可以直接使用，不用再次去DFS计算了。

002-如果相邻两个颜色均相同则删除当前颜色

这题就是找A或者B有连续三个以上的区域。比如说AAAA那么Alice就至少还有两次可以走的机会，BBBBB那么Bob就至少还有三次可以走的机会。那么我们的思路就是去计算Alice和Bob总共他们分别能够走几轮就再也不能走了。循环遍历这个colors字符串，每当碰到一个新的字符那么就开始新的一轮计数，如果这轮计数的结果 >= 3，那么就对应的加到总的能取的轮数当中。记NA为Alice总共能删的次数，NB为Bob总的能删的次数，那么如果NA > NB的话，则Alice胜，否则Bob胜。注意相等的时候也是Bob胜，因为是Alice走的先手。

003-网络空闲的时刻

这题题目的描述还挺复杂的，不过理解了之后就是一个最短路+不同情况的时间计算。首先是通过最短路算法计算server节点到每一个client节点的最短路径长，然后是遍历每个client节点，计算从该节点出发到server节点这条路径上没有网络包的最短时间。然后所有节点的最大值即为最后我们所需要的答案。

下一个问题就是要根据patience和distance之间的关系来计算每个节点的最短时间，记往返的总时间为dis。

  • 如果dis <= patience，那么时间就是dis

  • 如果dis % patience == 0，那么最后一个包发出去的时间是 (dis / patience - 1) * patience

  • 如果dis % patience != 0，那么最后一个包发出去的时间是 (dis / patience) * patience

对于后两种情况，再加上dis就是所需要计算的时间。不过需要注意的是，所有包都释放的时间的话，需要最后再加上1second。

004-两个有序数组的第 K 小乘积

这题之前确实没做过这种题，是做了一个转换，假设f(x) = nums1[i] * nums2[j] <= x的个数，那么我们的目标就是去寻找这么一个最小的x，使得刚好有k个乘积比这个数要小，所以这里可以用二分。

然后就是遍历第一个数组，然后在第二个数组当中进行第二次二分查找。假设在nums1当中的数n有三种情况：

  • n = 0: 如果target <= 0，那么合理的数就是nums2.size()

  • n > 0: 那么我们需要保证的是n *n2 <= target，也就是n2 <= floor(target / 1.* n)

  • n < 0: 那么我们需要保证n *n2 <= target，也就是n2 >= ceil(target / 1.* n)

这样进行二分查找，统计所有小于当前x的个数，根据总个数和k的关系调整left和right的值。不过这里需要注意一点的就是，这里left和right是有可能是负数的，所以最好用(left + right) >> 1来计算中间值，要不然可能会出现无限循环。

lower_bound: 第一个大于或等于x的数的位置
upper_bound: 第一个大于x的数的位置

周赛267

001-买票需要的时间

这道题就是直接循环就好了，然后碰到其他位置变成0就不用管，接着循环就好了。最后到自己这个位置减完之后，就只需要直接返回结果就好了。

002-反转偶数长度组的节点

这道题主要就是逻辑上比较复杂，在每个偶数的区间里面，都需要知道起始节点、终止节点、前驱节点和next节点。然后对这个区间的链表做一个链表反转。最后一个区间需要特殊考虑，因为也可能是偶数，总体就是链表反转的思路。

003-解密斜向换位密码

这道题我的思路是首先把这个矩阵给构造出来，然后把encodetext给按位填进去。填进去之后就按照斜对角线的方向将这个origintext给提取出来，就是有多少可能的位就提取多少位。
然后把最后得到的这个text的末尾的空格字符给全部去掉就是最后要得到的结果。

004-处理含限制条件的好友请求

这道题一看就是一道用并查集解决的问题。对于每个root节点都维护一个这个组里面的朋友和敌人。在一个好友请求到来的时候，首先会查看对应user所在的root组，如果两个人在一个组里，那就直接处理下一个好友请求。

否则的话，先要看看另一个组里的朋友列表有没有在该组的敌人列表里面出现。如果没有的话，说明可以进行union。否则的话，这个好友请求是不能通过的。

union的时候同时也把朋友列表和敌人列表同步更新。这里我们可以使用Bitset进行优化，因为最多也就是1000个user的数据量。

001-检查两个字符串是否几乎相同

这题就是简单的做一个check就好了。

002-模拟行走机器人ii

这题还有几个坑点的。首先是机器人是固定的沿着外圈走，所以有一个循环steps = 2 * width + 2 * (height - 2)。然后就是行进方向了，如果机器人是在(0, 0)开始的循环，那么最后回到(0, 0)并不是East方向，而是South方向。其他位置是木有这个special case的，然后其他的处理就是正常的处理好是否出界的情况就好。

003-每一个查询的最大美丽值

这题首先要对进来的items进行排序，首先是价格排序，如果价格相同那么就按美丽值从大到小进行排序。这个时候只需要对每个价格记录一下他们之前的最大美丽值就好。对每一个进来的query，我们就只需要通过二分的思路找到第一个小于等于当前query数的位置就好了，然后对应的美丽值就是我们想要的答案。
这里的二分的写法可以多看看，第一个小于等于的思路。
